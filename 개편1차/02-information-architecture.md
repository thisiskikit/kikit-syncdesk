# 02. Information Architecture

## 문서 목적
이 문서는 `KIKIT SyncDesk`의 상위 메뉴 구조와 화면 귀속 원칙을 재정의합니다.  
핵심 목표는 **채널 중심 IA를 액션 중심 IA로 재구성**하는 것입니다.

---

## 1) 최종 IA 원칙
### 원칙 A. 상위 메뉴는 업무 기준
상단 메뉴 또는 좌측 LNB는 **운영자가 무엇을 하려는가** 기준으로 구성합니다.

### 원칙 B. 채널은 세부 레이어
쿠팡/네이버는 중요하지만, 상위 네비게이션의 주인공이 아닙니다.  
채널은 필터, 세부 원본 확인, 연결 설정, 채널별 고급 제어 기능의 레이어로 이동합니다.

### 원칙 C. 핵심 화면은 1차 업무 흐름 중심
초기 진입은 아래 흐름을 우선시합니다.

- 오늘 운영 확인
- 출고 판단 및 송장 처리
- CS 영향 확인
- 실패 작업 복구
- 채널 원본 확인

---

## 2) 최종 상위 메뉴
최종 상위 메뉴는 아래 6개로 고정합니다.

1. 대시보드
2. 출고
3. CS
4. 채널
5. 작업센터
6. 설정

---

## 3) 각 메뉴의 목적

## 대시보드
### 목적
오늘 처리할 일의 우선순위를 한눈에 보여주고, 바로 작업 화면으로 진입시키는 시작점

### 포함 요소
- 오늘 출고
- 출고 차단
- 송장 대기
- 답변 필요
- 실패 작업
- 최근 이슈 피드
- 빠른 작업 버튼

---

## 출고
### 목적
수집된 주문을 모아서 **출고 가능 여부를 판단**하고 송장을 반영하는 핵심 작업 공간

### 포함 요소
- 출고 상태 탭
- 채널/날짜/판단 필터
- 메인 테이블
- 우측 Drawer
- 일괄 작업 바
- 송장 입력 액션

---

## CS
### 목적
모든 채널의 클레임과 문의를 한곳에서 처리하되, 출고 판단에 미치는 영향을 함께 보여주는 공간

### 포함 요소
- 답변 필요
- 취소/반품/교환
- 출고중지
- 출고 영향 있음
- 이력 검색

---

## 채널
### 목적
쿠팡, 네이버 등 연결 상태와 채널별 원본 데이터, API 세팅, 채널 고유 제어 기능을 확인하는 공간

### 포함 요소
- 쿠팡 연결 및 세부
- 네이버 연결 및 세부
- 채널별 원본 데이터 조회
- 상품/제어/정산/통계
- 세부 로그 및 연동 정보

---

## 작업센터
### 목적
수집/동기화/반영/송장 작업의 성공·실패 로그를 확인하고 실패 작업을 복구하는 공간

### 포함 요소
- 실행 중
- 최근 완료
- 실패/경고
- 재시도 필요
- 작업 로그

---

## 설정
### 목적
기본 정책, 연결 상태, UI 상태, 관리자 옵션을 관리하는 공간

### 포함 요소
- 채널 연결 설정
- 기본 정책
- 작업 기본값
- UI 상태
- 관리자 기능

## 5) 라우트 재배치 원칙
### 원칙 A. 구 경로는 바로 삭제하지 않는다
기존 라우트는 사용 습관과 QA 충격을 고려해 즉시 제거하지 않습니다.

### 원칙 B. 새 IA 기준으로 재귀속한다
예:
- `/coupang/shipment` → `/fulfillment/today?channel=coupang`
- `/coupang/claims` → `/cs/claims?channel=coupang`
- `/naver/inquiries` → `/cs/inbox?channel=naver`
- `/runs` → `/operations/recent`

### 원칙 C. Nav는 새 구조만 노출
실사용자가 보는 navigation에는 새 구조만 보여주고, 기존 라우트는 필요 시 redirect 또는 wrapper로 유지합니다.

## 7) 페이지 간 이동 원칙
### 대시보드 → 출고
- 오늘 출고 카드 클릭 → 출고 화면 `출고 가능` 또는 `오늘 출고` 필터
- 출고 차단 카드 클릭 → 출고 화면 `차단됨`
- 송장 대기 카드 클릭 → 출고 화면 `송장 대기`

### 대시보드 → CS
- 답변 필요 카드 클릭 → CS 화면 `답변 필요`

### 대시보드 → 작업센터
- 실패 작업 카드 클릭 → 작업센터 `실패/경고`

### 출고 ↔ CS
- 출고 화면 row 또는 Drawer에서 CS 영향 이력 클릭 → CS 상세 컨텍스트로 이동

### 출고 ↔ 채널
- Drawer 내 원본 데이터/채널 세부 보기 클릭 → 채널 화면 또는 원본 모달/Drawer

---

## 8) Breadcrumb 원칙
Breadcrumb는 기술 구조가 아니라 업무 흐름을 반영해야 합니다.

### 예시
- 대시보드
- 출고 > 오늘 출고
- 출고 > 차단됨
- CS > 답변 필요
- 채널 > 쿠팡 > 원본 데이터
- 작업센터 > 실패/경고

---

## 9) IA 성공 기준
- 상단 메뉴를 보는 순간 제품이 “운영 데스크”로 느껴진다.
- 쿠팡/네이버는 주 메뉴가 아니라 세부 업무 레이어로 느껴진다.
- 대시보드, 출고, CS, 작업센터가 실무 행동 중심으로 연결된다.
- 레거시 상품/가격 기능이 메인 정체성을 더 이상 방해하지 않는다.
