# 03. Dashboard Redesign

## 문서 목적
이 문서는 `KIKIT SyncDesk`의 Dashboard를 **분석용 홈 화면**이 아니라 **오늘의 운영 시작점**으로 재설계하기 위한 상세 기준을 정의합니다.

---

## 1) 대시보드의 역할 재정의
### 잘못된 해석
- KPI 카드가 많은 요약판
- 각 채널 상태를 넓게 보여주는 홈
- 보기 좋은 숫자 모음

### 올바른 해석
**대시보드는 오늘 처리해야 할 운영 액션을 우선순위대로 보여주고, 바로 해당 작업 화면으로 이동시키는 실행 런치패드입니다.**

---

## 2) 최우선 질문
대시보드는 아래 질문에 답해야 합니다.

> 지금 내가 무엇부터 처리해야 하는가?

대시보드는 정보를 다 보여주는 화면이 아니라,  
운영자가 **오늘 첫 클릭**을 어디에 해야 하는지 안내하는 화면이어야 합니다.

---

## 3) 화면 구조
대시보드는 아래 5개 블록으로 구성합니다.

1. 헤더 영역
2. 핵심 액션 카드
3. 우선순위 패널
4. 최근 이슈 피드
5. 빠른 작업 패널
6. 하단 상태 스트립

---

## 4) 블록별 상세 설계

## A. 헤더 영역
### 구성
- 페이지 제목: `오늘 운영`
- 보조 문구: `지금 바로 처리해야 할 출고, CS, 실패 작업을 먼저 보여드립니다.`
- 우측 버튼:
  - `빠른 수집`
  - `작업센터 열기`

### 의도
첫 화면에서 제품 성격을 설명하는 동시에 바로 행동을 시작하게 합니다.

---

## B. 핵심 액션 카드
### 카드 구성 원칙
각 카드는 다음 3요소를 반드시 포함합니다.

- 현재 건수
- 건수의 의미를 설명하는 한 줄
- 클릭 시 연결되는 명확한 CTA

### 카드 1. 오늘 출고
- 타이틀: `오늘 출고`
- 설명: `지금 출고 가능한 주문`
- CTA: `출고 열기`
- 연결: 출고 화면 > `오늘 출고`

### 카드 2. 출고 차단
- 타이틀: `출고 차단`
- 설명: `클레임/문의/상태 이슈로 멈춘 주문`
- CTA: `차단 건 보기`
- 연결: 출고 화면 > `차단됨`

### 카드 3. 송장 대기
- 타이틀: `송장 대기`
- 설명: `출고 전 송장 입력이 필요한 주문`
- CTA: `송장 입력`
- 연결: 출고 화면 > `송장 대기`

### 카드 4. 답변 필요
- 타이틀: `답변 필요`
- 설명: `문의 및 CS 응답이 필요한 건`
- CTA: `CS 열기`
- 연결: CS 화면 > `답변 필요`

### 카드 5. 실패 작업
- 타이틀: `실패 작업`
- 설명: `수집/반영/동기화 실패 건`
- CTA: `작업센터 보기`
- 연결: 작업센터 > `실패/경고`

### 카드 배치 원칙
- Desktop: 5개 1행 또는 3+2 배치
- 좁은 화면: 2행
- 카드 수는 5개를 넘기지 않음

---

## C. 우선순위 패널
### 제목
`지금 먼저 볼 것`

### 표시 형식
리스트형 또는 우선순위 카드형

### 예시
1. 출고 차단 12건
2. 송장 대기 8건
3. 답변 지연 위험 3건
4. 발송 후 이슈 2건

### 의도
숫자 나열이 아니라 **행동 순서**를 제안합니다.

### 클릭 규칙
각 항목 클릭 시 해당 필터 상태가 적용된 목적 화면으로 이동합니다.

---

## D. 최근 이슈 피드
### 제목
`최근 이슈`

### 포함 이벤트
- 새 클레임 유입
- 송장 반영 실패
- 발송 후 이슈
- 전체 수집 완료 / 경고
- 채널 API 오류
- 상태 충돌 감지

### 표시 방식
- 리스트형 또는 타임라인형
- 5~7개까지만 표시
- 클릭 시 작업센터, 출고, CS, 채널 상세로 이동

### 의도
작업센터에 들어가기 전에도 운영자가 최근에 무슨 일이 일어났는지 즉시 파악하게 합니다.

---

## E. 빠른 작업 패널
### 구성 예시
- 빠른 수집
- 송장 입력 열기
- 차단 건 보기
- 실패 작업 보기

### 원칙
- 단축 작업만 배치
- 3~5개 이내
- 레거시 기능 버튼 금지

---

## F. 하단 상태 스트립
### 표시 예시
- 쿠팡 연결 정상
- 네이버 연결 정상
- 마지막 수집 10:32
- 마지막 실패 09:58

### 의도
채널 상태는 중요하지만 메인이 아니므로 큰 카드가 아니라 얇은 스트립으로 처리합니다.

---

## 5) 시각적 우선순위
### 가장 먼저 보이는 것
- 오늘 출고
- 출고 차단
- 송장 대기

### 다음으로 보이는 것
- 답변 필요
- 실패 작업
- 우선순위 패널

### 가장 뒤에 가야 하는 것
- 채널 세부 상태
- 긴 로그
- 원본 데이터
- 통계성 보조 수치

---

## 6) 빈 상태 문구
### 오늘 출고 카드
`지금 바로 처리할 출고 건이 없습니다.`

### 출고 차단 카드
`현재 차단된 주문이 없습니다.`

### 송장 대기 카드
`송장 입력이 필요한 주문이 없습니다.`

### 답변 필요 카드
`답변이 필요한 문의가 없습니다.`

### 실패 작업 카드
`현재 실패한 작업이 없습니다.`

### 최근 이슈 피드
`최근 이슈가 없습니다.`

---

## 7) 대시보드 액션 흐름
### 흐름 1
`오늘 출고` 카드 클릭  
→ 출고 화면 `오늘 출고` 필터 활성화  
→ 테이블 row 선택  
→ Drawer 열림

### 흐름 2
`출고 차단` 카드 클릭  
→ 출고 화면 `차단됨`  
→ 차단 사유 확인  
→ 필요한 경우 CS로 이동

### 흐름 3
`실패 작업` 카드 클릭  
→ 작업센터 `실패/경고`  
→ 재시도 가능 작업 확인

---

## 8) 금지사항
- 채널별 큰 카드 나열 금지
- bulk-price/product-edit 바로가기 금지
- 원본 데이터 중심 홈 화면 금지
- 의미 없는 KPI 과다 노출 금지
- 카드만 예쁘고 CTA가 약한 구조 금지

---

## 9) 성공 기준
- 대시보드를 보는 순간 오늘 어떤 작업이 중요한지 알 수 있다.
- 카드가 숫자 요약이 아니라 작업 시작점으로 기능한다.
- 최근 이슈와 우선순위가 살아 있어 운영자가 머뭇거리지 않는다.
- 제품이 “운영 데스크”처럼 느껴진다.
