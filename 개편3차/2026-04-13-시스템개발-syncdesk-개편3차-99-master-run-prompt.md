# 실행용 마스터 프롬프트 — SyncDesk 개편 3차

아래 프롬프트를 Codex에 그대로 넣고 실행합니다.

---

`kikit-syncdesk` 저장소의 `codex/dev` 브랜치 기준으로 개편 3차 작업을 진행하라.

현재 작업 폴더에 `개편3차` 폴더가 있으며, 그 안에 이번 작업용 MD 문서들이 들어 있다.
이번 작업의 목적은 **운영 연결 완성 + 운영 하드닝**이다.

반드시 아래 MD 파일들을 먼저 읽고, 순서대로 작업하라.

읽을 파일 순서:
1. `개편3차/2026-04-13-시스템개발-syncdesk-개편3차-00-common-rules.md`
2. `개편3차/2026-04-13-시스템개발-syncdesk-개편3차-01-ops-linking.md`
3. `개편3차/2026-04-13-시스템개발-syncdesk-개편3차-02-hub-cleanup.md`
4. `개편3차/2026-04-13-시스템개발-syncdesk-개편3차-03-hardening.md`

실행 방식:
- 1번 파일의 공통 운영 규칙을 모든 작업에 적용한다.
- 2번 작업을 먼저 수행하고 결과를 보고한다.
- 3번 작업을 이어서 수행하고 결과를 보고한다.
- 4번 작업을 마지막으로 수행하고 결과를 보고한다.
- 각 단계에서 코드 변경과 docs 변경을 반드시 함께 처리한다.
- 검증하지 못한 내용은 `추정`으로 표시한다.
- 기존 동작을 깨지 않는 것을 우선한다.
- 작업을 임의로 다른 범위로 넓히지 않는다.

이번 차수의 허용 범위:
- 출고-작업센터-CS 연결 흐름 강화
- 채널/설정 허브 정리 및 레거시 진입 구조 정돈
- 성능/배포/운영 하드닝

이번 차수의 금지 범위:
- 상품/가격 기능 대확장
- CS 전체 신규 백엔드 시스템 구축
- 모바일 완성형 대응
- 전면적인 디자인 토큰 재설계
- unrelated page 리디자인
- 대규모 DB 스키마 변경

4. 테스트 결과
