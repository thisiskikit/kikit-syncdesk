# 개편 3차-2 — 채널 / 설정 허브 정리와 레거시 진입 구조 정돈

## 작업 목적
`kikit-syncdesk`의 `codex/dev` 기준으로, 채널/설정 허브를 더 명확한 진입점으로 정리하고
레거시 도구 진입 구조를 operator 관점에서 덜 헷갈리게 만듭니다.

## 반드시 먼저 읽을 문서
- `2026-04-13-시스템개발-syncdesk-개편3차-00-common-rules.md`
- `docs/current-status.md`

## 작업 목표
채널과 설정 화면이 “어디로 들어가야 하는지 모호한 허브”가 아니라
**연결/원본 확인/고급 도구 진입점**으로 명확히 읽히게 만듭니다.

## 반드시 달성할 것
1. 채널 허브의 정보구조를 더 분명히 정리합니다.
   - 연결 상태
   - 원본 화면 진입
   - 채널별 주요 도구
2. 설정 허브의 정보구조를 더 분명히 정리합니다.
   - 연결 설정
   - 고급 도구
   - 레거시 진입
3. 레거시/고급 도구는 계속 지원하되 메인 운영 흐름을 방해하지 않게 배치합니다.
4. 카드/설명/empty state 문구를 운영 문장으로 정리합니다.
5. 기존 route와 deep-link는 유지합니다.
6. 관련 tests 와 docs 를 함께 갱신합니다.

## 권장 구현 방향
- 허브를 “채널별 정보 모음”이 아니라 “무엇을 하려는가” 기준으로 더 재구성
- 연결 상태와 원본 화면 이동을 더 상단에 배치
- 레거시 도구는 고급/기타 섹션으로 명확히 분리
- 설정 문구를 operator가 이해하는 언어로 다듬기

## 하지 말 것
- 채널 세부 페이지 전면 재구축
- 신규 상품/가격 도구 추가
- unrelated settings redesign
- route 삭제 위주의 과격한 정리

## 완료 기준
- 채널/설정 허브가 더 빠르게 읽힙니다.
- operator가 들어갈 위치를 덜 고민합니다.
- 레거시 도구는 남기되 주동선을 방해하지 않습니다.
- docs가 함께 반영됩니다.

4. 테스트 결과
