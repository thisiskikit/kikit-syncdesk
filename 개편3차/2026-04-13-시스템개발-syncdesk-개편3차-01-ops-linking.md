# 개편 3차-1 — 출고 / CS / 작업센터 연결 흐름 강화

## 작업 목적
`kikit-syncdesk`의 `codex/dev` 기준으로, 출고와 작업센터와 CS 허브가 서로 떨어진 화면이 아니라
**운영자가 자연스럽게 이어서 쓰는 연결된 흐름**처럼 보이게 만듭니다.

## 반드시 먼저 읽을 문서
- `2026-04-13-시스템개발-syncdesk-개편3차-00-common-rules.md`
- `docs/current-status.md`
- `docs/decisions/2026-04-12-syncdesk-operation-desk-ia.md`

## 작업 목표
출고 화면에서 예외/재확인/차단 상태를 봤을 때,
operator가 다음 액션으로 `CS` 또는 `작업센터`로 이어질 수 있게 연결을 강화합니다.

## 반드시 달성할 것
1. 출고 화면에서 CS 영향 또는 후속 조치 필요성을 더 명확히 드러냅니다.
2. 차단 / 재확인 필요 / 보류 상태에서 다음 액션 경로를 더 분명하게 보여줍니다.
3. 작업센터 상세와 출고 상세 사이의 연결 문맥을 강화합니다.
4. CS 허브는 여전히 허브로 유지하되, deep-link 목적성과 안내 문구를 더 분명하게 만듭니다.
5. 사용자가 “지금 어디로 가야 하는지”를 더 빨리 판단할 수 있게 연결 버튼/문구/요약을 정리합니다.
6. 기존 deep-link 구조와 route 호환은 유지합니다.
7. 관련 테스트와 docs를 함께 갱신합니다.

## 권장 구현 방향
- 출고 drawer 또는 요약 영역에서 후속 액션 힌트 강화
- 작업센터 상세 패널에 연관 화면 이동 포인트 정리
- CS 허브 카드/설명 문구를 operator 맥락 중심으로 재정리
- “문의/클레임 확인 필요”, “복구 작업 먼저”, “채널 원본 확인” 같은 운영 문장을 명확히 적용

## 하지 말 것
- CS 전체 신규 실행 시스템 구축
- 채널별 문의 로직 전면 재설계
- 대규모 백엔드 워크플로우 추가
- unrelated page 리디자인

## 완료 기준
- 예외 주문을 본 뒤 다음 이동 경로가 더 분명합니다.
- 출고/작업센터/CS 허브의 역할이 충돌하지 않습니다.
- deep-link는 유지되며 operator 판단 시간이 줄어듭니다.
- 테스트와 docs가 함께 반영됩니다.

## 최종 보고 형식

2. 연결 흐름이 어떻게 바뀌었는지
3. 유지된 동작 / 달라진 안내 구조
4. 테스트 결과
5. 직접 검증 못 한 것
6. 남은 위험
