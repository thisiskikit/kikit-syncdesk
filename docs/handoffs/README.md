# Handoffs

Use this directory for task handoff notes when work cannot be finished in one pass or when a follow-up owner needs exact context.

## Suggested File Naming

- `YYYY-MM-DD-short-title.md`

## Recommended Contents

- What changed
- Why it changed
- Impact scope
- Current blockers
- Remaining issues
- Next work
- Verification performed
- Verification not performed
- Open questions

## Writing Rules

- Mark anything unverified as `추정`.
- If code changed before the handoff, keep the documentation set in sync:
  - `docs/current-status.md`
  - `docs/change-log.md`
  - `docs/structure-overview.md`
