# Manual Fulfillment Regression

- 기준 날짜: 2026-04-13
- 기준 브랜치: `codex/dev`
- 목적: 현재 출고 운영 화면의 핵심 회귀 시나리오를 수동 검증 절차로 고정합니다.
- 주의: 브라우저에서 아직 직접 검증하지 못한 기대 결과는 `추정`으로 표시합니다.

## 사전 조건

- 최소 1개 이상의 쿠팡 스토어가 연결되어 있어야 합니다.
- `출고 가능`, `차단`, `재확인 필요` 상태가 섞인 샘플 row가 있으면 좋습니다.
- `빠른 수집(new_only)`에서 신규 주문이 실제로 추가되는 환경이면 가장 좋습니다.

## 시나리오 1. dashboard -> fulfillment 진입

- 위험도: medium
- 단계:
1. `/dashboard`에 진입합니다.
2. 대시보드의 `출고` 진입 카드 또는 동일 의미 CTA를 클릭합니다.
3. `/fulfillment` 화면이 열리는지 확인합니다.
- 기대 결과:
  - 상단 hero에 `출고` 제목과 운영 설명이 보입니다.
  - `작업 화면 / 보관함 / 화면 설정` segmented control이 보입니다.
  - 기본 필터, 출고 판단 요약, 메인 grid가 한 화면 흐름으로 이어집니다.

## 시나리오 2. 빠른 수집 후 신규 주문 집중 보기

- 위험도: high
- 단계:
1. `작업 화면`에서 스토어를 선택합니다.
2. `빠른 수집`을 실행합니다.
3. 수집 완료 후 상단 요약과 grid를 확인합니다.
4. `출고 판단` 탭 또는 보기 범위를 바꿉니다.
5. 다시 기본 상태로 돌아와 `전체 보기로 돌아가기` 또는 집중 보기 해제 동작을 확인합니다.
- 기대 결과:
  - 신규 주문이 실제로 추가된 경우, 요약에 “빠른 수집으로 추가된 N건을 먼저 보여줍니다” 메시지가 보입니다.
  - 집중 보기 중에도 출고 판단 탭은 유지됩니다.
  - 필터 시그니처가 바뀌면 집중 보기가 해제되고 원래 시트 기준 목록으로 복귀합니다.
  - `추정`: 신규 주문이 없으면 집중 보기 메시지는 보이지 않고 기존 목록이 유지됩니다.

## 시나리오 3. 혼합 선택 후 자동 제외 실행

- 위험도: high
- 단계:
1. `출고 가능` row와 `차단` 또는 `재확인 필요` row를 함께 선택합니다.
2. 선택 bar의 요약 문구를 확인합니다.
3. `송장 전송` 또는 `선택 행 엑셀 다운로드`를 실행합니다.
- 기대 결과:
  - 선택 bar에 제외 대상 요약이 먼저 보입니다.
  - 실행 가능한 주문이 하나라도 있으면 실행 버튼은 유지됩니다.
  - 제외 대상은 자동으로 빠지고, 실행 가능한 주문만 처리됩니다.
  - `추정`: 실제 네트워크 실행 결과도 제외 대상은 payload에 포함되지 않아야 합니다.

## 시나리오 4. fulfillment drawer 확인 흐름

- 위험도: medium
- 단계:
1. 편집 셀이 아닌 일반 row 영역을 클릭합니다.
2. 우측 Drawer가 열리는지 확인합니다.
3. Drawer 안의 `전체 상세` 진입을 클릭합니다.
4. 전체 상세 dialog를 닫고 다시 drawer 또는 목록으로 복귀합니다.
- 기대 결과:
  - 첫 클릭에서는 전체 페이지 전환이 아니라 drawer가 먼저 열립니다.
  - drawer에는 출고 판단, 상태, 배송, 활동 정보가 보입니다.
  - 전체 상세 dialog에서는 claim/return/exchange와 주문 항목 정보가 추가로 보입니다.
  - 닫기 후에는 `/fulfillment` route를 유지한 채 원래 목록 문맥으로 돌아옵니다.

## High-Risk 회귀 포인트

- high: 빠른 수집 집중 보기 활성/해제 조건이 필터 변경과 충돌하는지
- high: 혼합 선택에서 차단 row가 payload에 섞여 들어가지 않는지
- high: drawer와 전체 상세 dialog가 같은 detail row 문맥을 유지하는지
- medium: 보관함/화면 설정 탭으로 이동해도 상단 action 비활성화 규칙이 맞는지
- low: 요약/메트릭 문구가 현재 필터 상태와 어긋나지 않는지

## 자동화 테스트 후보

- `빠른 수집 응답 -> quickCollectFocus 활성 -> 필터 변경 -> 자동 해제` 흐름을 component/integration test로 고정
- 혼합 선택 시 실행 payload에서 blocked row가 빠지는지 action helper test로 고정
- drawer open -> full detail open -> close 복귀 흐름을 browser automation으로 검증
