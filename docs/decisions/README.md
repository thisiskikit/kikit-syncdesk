# Decisions

Use this directory for design and architecture decisions that affect code structure, persistence, API ownership, workflow boundaries, or operational behavior.

## Suggested File Naming

- `YYYY-MM-DD-short-title.md`

## Recommended Contents

- Context
- Decision
- Reason
- Impact scope
- Alternatives considered
- Remaining issues
- Next work
- Verification status

## Writing Rules

- Mark anything unverified as `추정`.
- Do not mark untested changes as complete.
- If the decision changes code behavior, also update:
  - `docs/current-status.md`
  - `docs/change-log.md`
  - `docs/structure-overview.md`
