# 개편 2차-2 — 작업센터 recovery-first 강화

## 작업 목적
`kikit-syncdesk`의 `codex/dev` 기준으로 작업센터를 단순 로그 화면이 아니라 **복구 중심 운영 콘솔**로 더 선명하게 만듭니다.

## 반드시 먼저 읽을 문서
- `2026-04-13-시스템개발-syncdesk-개편2차-00-common-rules.md`
- `docs/current-status.md`
- `docs/decisions/2026-04-12-syncdesk-operation-desk-ia.md`

## 작업 목표
작업센터 첫 화면에서 operator가 “무슨 문제가 있었는지”보다 “지금 무엇을 복구해야 하는지”를 먼저 알 수 있게 구조를 다듬습니다.

## 반드시 달성할 것
1. 실패 / 경고 / 건너뜀 / 성공 우선순위 표현을 더 선명하게 합니다.
2. 재시도 가능 여부를 리스트 상단 판단 요소로 끌어올립니다.
3. 실패 원인을 operator 관점에서 빠르게 읽히도록 그룹핑/요약을 개선합니다.
4. 상세 패널 helper 또는 section 책임을 분리합니다.
5. 조치 유도 문구를 강화합니다.
6. 관련 테스트를 보강합니다.
7. 아래 문서를 함께 갱신합니다.
   - `docs/current-status.md`
   - `docs/change-log.md`
   - `docs/structure-overview.md`

## 개선 방향
- 복구 가능 작업을 더 먼저 보이게 합니다.
- 동일 성격 실패를 묶어 operator 판단 비용을 줄입니다.
- 주문 단위 영향 범위를 더 빠르게 읽히게 합니다.
- 상세 패널은 설명보다 조치 우선 흐름으로 정리합니다.

## 하지 말 것
- 완전히 새로운 백엔드 워크플로우 도입
- CS 통합 실행 화면 확장
- unrelated page 리디자인
- 로그 정보를 과하게 추가해 오히려 판단을 늦추는 개편

## 완료 기준
- 작업센터가 “복구할 일” 중심으로 읽힙니다.
- 리스트에서 재시도 가능 여부가 더 먼저 보입니다.
- 상세 패널은 행동 유도 중심입니다.
- 테스트와 docs가 함께 반영됩니다.

## 최종 보고 형식
1. 변경한 파일 목록
2. 작업센터 정보구조 변화
3. 복구 동선이 어떻게 개선됐는지
4. 테스트 결과
5. 직접 검증 못 한 것
6. 남은 리스크
7. docs 반영 내용
