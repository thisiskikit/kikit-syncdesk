# 실행용 마스터 프롬프트 — SyncDesk 개편 2차

아래 프롬프트를 Codex에 그대로 넣고 실행합니다.

---

`kikit-syncdesk` 저장소의 `codex/dev` 브랜치 기준으로 개편 2차 작업을 진행하라.

이번 작업은 새 기능 추가가 아니라 **운영 구조 고정**이 목적이다.  
반드시 아래 MD 파일들을 먼저 읽고, 순서대로 작업하라.

읽을 파일 순서:
1. `2026-04-13-시스템개발-syncdesk-개편2차-00-common-rules.md`
2. `2026-04-13-시스템개발-syncdesk-개편2차-01-fulfillment-refactor.md`
3. `2026-04-13-시스템개발-syncdesk-개편2차-02-work-center-recovery.md`
4. `2026-04-13-시스템개발-syncdesk-개편2차-03-manual-regression-docs.md`

실행 방식:
- 1번 파일의 공통 운영 규칙을 모든 작업에 적용한다.
- 2번 작업을 먼저 수행하고 결과를 보고한다.
- 3번 작업을 이어서 수행하고 결과를 보고한다.
- 4번 작업을 마지막으로 수행하고 결과를 보고한다.
- 각 단계에서 코드 변경과 docs 변경을 반드시 함께 처리한다.
- 검증하지 못한 내용은 `추정`으로 표시한다.
- 기존 동작을 깨지 않는 것을 우선한다.
- 최종 응답에서는 작업 1/2/3 각각에 대해 아래 형식으로 정리한다.

최종 응답 형식:
# 작업 1
1. 변경한 파일 목록
2. 구조 변경 요약
3. 테스트 결과
4. 직접 검증 못 한 것
5. 남은 위험
6. docs 반영 내용

# 작업 2
1. 변경한 파일 목록
2. 정보구조/복구 흐름 변경 요약
3. 테스트 결과
4. 직접 검증 못 한 것
5. 남은 위험
6. docs 반영 내용

# 작업 3
1. 생성/변경 문서 목록
2. 핵심 시나리오 요약
3. high-risk 회귀 포인트
4. 자동화 테스트 후보
5. docs 반영 내용

주의:
- CS 통합 실행 화면 신설 금지
- 채널 세부 화면 전면 개편 금지
- 상품/가격 기능 재확장 금지
- 디자인 토큰 대공사 금지
- 모바일 완성형 대응 금지

작업을 임의로 다른 범위로 넓히지 말고, 위 MD 파일 범위 안에서만 진행하라.
